Web Storage

Web Storage 用来给浏览器提供一种更直观的存储键值对的机制。

包含： Window.sessionStorage、 Window.lacalStorage

  1. Window.sessionStorage
    • 提供对当前域的会话访问
    • 该存储区域在页面会话期间可用（浏览器在打开状态，包括重新加载和恢复）
  2. Window.lacalStorage
    • 提供本地Storage对象的访问
    • 该存储区域存储数据会保留在浏览器中

cookie

与localStorage常用来做对比的对象是：HTTP Cookie（也叫Web cookie或浏览器Cookie）
cookie是为了方便 保存http的状态信息 而产生的（http是无状态协议），它是服务器发送到用户浏览器并保存在浏览器上的一块数据，在发送http请求时在 浏览器和服务器之间来回传送 。
cookie常用于确定两次请求是否来自于同一个浏览器，从而能够确认和保持用户的登录状态。
在Web应用中，cookie常用来 标记用户 和 会话授权。也正因为这样，窃取cookie可以进行 会话劫持 或利用程序的漏洞进行 XSS攻击、 跨站请求伪造（CSRF），存在着网络安全隐患

对比 访问对象 过期时间 大小 同源窗口是否共享 是否发送
sessionStorage 当前域的会话 浏览器关闭时 5M内 不共享（待验证） 仅本地存储
localStorage 本地Storage对象 没有 5M内 共享 仅本地存储
cookie 来自服务器 存在，可设置 4k 共享 在http请求中携带
