跨域问题入门篇

Veröffentlicht am 2017-03-17

跨域问题已经困扰我很久了，因为时不时总会在各种推送中看到这个问题，然而它到底是什么？仿佛所有的文章一提到跨域就会说相关的一些词汇，如“同源”，“JSONP”，“AJAX”……
然后我看完，还是觉得到底 什么是跨域啊！！！

最近我终于，终于在自己的项目里遇到了跨域问题，好激动啊，终于有个机会来了解下跨域了。

当我们在浏览网页的时候，最常做的操作就是点击，点击之后，浏览器会给出相应的反馈，比如跳转到另外一个页面……

好，那么这个时候问题来了，

  • 第二个页面怎么来的？

其实在我们的点击背后附加这一个链接，而这个链接就是第二个页面的地址，当然这个大家都知道，关键在于 浏览器本身并不会存储这第二个页面，它只能 向服务器去请求这个页面。

  • 怎么请求的呢？

浏览器向服务器发送了一个 http请求，请注意是http请求，也就是Http Request，有点眼熟，我们还知道一个JavaScript对象叫做XMLHttpRequest，而AJAX正是封装了这个XMLHttpRequest对象实现了异步交互数据的功能。。。等等，我是不是发现了什么？

  • 言归正传，服务器收到http请求后呢？

服务器把你要的 页面返回给浏览器，然后浏览器把 页面渲染在你的眼前，完事。

当然事实上不会这么轻易的就返回给你的，浏览器得判断一下你有没有资格来拿这个页面，所谓 资格，简单来说，就是判断一下你（当前页面）和请求页面是不是一家的。判断标准就是你俩的域名(domain)，协议(protocol)，端口(port) 是不是一样的，一样的，你拿去，不一样，那对不起，你没有资格拿到这个页面。对，判断标准就是同源策略。当你拿不到这个页面的时候，就是出现了跨域的问题了。

好，废话这么多，总结一下：

  1. 跨域问题总是会出现在浏览器向服务器请求数据的时候
  2. 同源策略决定你需不需要跨域
  3. AJAX总是出现在你向服务器要数据的时候，所以有些情况下AJAX和跨域扯到一起

以上就是一个前端小白，对“跨域是什么”这个问题纠结的理解过程。。。

js中的遍历和查找

Veröffentlicht am 2017-03-13

对计算式进行处理，其实就是对字符串进行遍历，然后再按照规则进行匹配的过程。
在做 计算器 的过程中，我先后遇到了之前处理问题时不曾考虑过的情况：

  1. 遍历数组用哪个？
  2. 为什么我写的循环break不能正确响应？
  3. 明明只需要查找一个元素，却要要把整个数组都遍历一次，代价太大了吧？

其实，js提供了很多很好用的接口，只是平时的一些编码习惯使我们在遇到某种情况时，下意识的的就只能反应到一种或两种常用方法，但这种方法却不一定是当前最合适的。
书到用时方恨少，方法也是这样。js的各种API才是提高编码效率的利器啊！
借这个机会，我认真梳理了js中常见的遍历和查找方法。现将常用的遍历和查找方法做了简单的对比，如下：

遍历函数

for()循环

主要用于数组遍历
按序传入index，定位到需要操作的元素
可以通过break，continue，return打断

forEach()

Array.prototype.forEach()
按序传入：当前值，索引，数组
不支持打断，只能用异常处理，将forEach代码块包在try语句块中
若循环过程中数组发生了删除操作，某些项目可能被forEach忽略

for … in

以任意序迭代对象的可枚举属性名（所有可枚举属性和继承可枚举属性）
不应该用来迭代一个下标顺很重要的数组，因为for … in 是无序的
同时，循环的属性名为字符串
若循环过程中对象发生了增、删、改操作，for … in 不保证得到期望结果
使用 break 和continue 打断

  • 若只迭代对象本身的属性
    使用 getOwnPropertyNames() 或hasOwnProperty()
  • 属性是否可枚举
    使用 obj.propertyIsEnumerable()

for … of

在可迭代对象上创建循环，遍历当前对象上的属性值
可迭代对象：Array、 Map、 Set、 String、 arguments等
推荐在遍历数组时使用
支持打断 break、 continue

map()

Array.prototype.map()
通过 in polyfill
创建一个新的数组，存放数组中每个元素执行函数后的结果

every()

Array.prototype.every()
通过 for循环 polyfill
意在测试数组中所有元素是否都通过了测试 返回 布尔值
可以被打断 return
相关函数 some() 有一个元素满足条件即为TRUE
filter() 过滤函数

jQuery.each() === $.each()

遍历数组和对象
return 打断

$(selector).each()

遍历dom元素
return 打断

jQuery也有map方法

匹配查找方法

in

严格来说 in 并不是遍历方法，主要用于查找
当遍历的目的在于查找时，使用in会比循环更高效一些，
右操作数必须为 包装对象 Array Object String
原型链上的属性也会返回true

has()

Map、 Set 对象方法。是否存在指定元素
传入 键值(value)

indexOf()

String、 Array对象方法。 返回查找元素的 index值

find(callback)

Array对象方法。 返回满足callback的第一个元素的索引

findIndex(callback)

Array对象方法。 返回满足callback的第一个元素的索引

includes(value)

String、 Array对象方法。 判断数组是否包含当前值

后续有其他的相似功能的方法会继续添加进来的~~

后缀表达式是二叉树后序遍历的结果

转化过程

  • 借助二叉树图形化理解过程
    其实中缀表达式转后缀表达式就是同一棵二叉树由中序遍历结果转化为后序遍历结果
    单单只有中序遍历结果是不能得到唯一的后序遍历结果的
    所以在处理时，要对操作符的优先级进行设置，得到唯一的遍历结果

  • 具体转化步骤
    在转化之前需明确：
    输入： 中缀表示式 > inputStr
    输出： 后缀表达式 > outputStr
    工具： 计算机调用栈 > stack
    步骤：

    1. 对输入串进行遍历，每次得到input[i]，对每个input[i]进行判断
2. 若input[i]为数字，添加到outputStr
3. 若input[i]为左括号，添加到stack
4. 若input[i]为右括号，将stack中左括号之前的操作符添加到outputStr,左括号出栈
5. 若input[i]为操作符，
   当出现下面三种情况之一时，直接添加到stack
   - stack 为空
   - stack 栈顶 为 左括号
   - 当前input[i]的优先级高于栈顶操作符优先级
   其他情况，先将栈顶元素出栈后，再将input[i]添加到stack
6. 遍历结束
7. 若stack中仍有操作符，出栈，添加到outputStr
